# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/),
проект придерживается [Semantic Versioning](https://semver.org/).

## [0.3.0] - 2026-04-14

### Added

- Бот понимает голосовые сообщения и круглые видео. Задай вопрос голосовой с @упоминанием или в ответ к боту — он транскрибирует через OpenAI Whisper и ответит как на обычный текст. Плюс бот может внезапно прокомментировать чужое голосовое в чате, если сработает кубик интерджекта.
- Reply-chain: бот понимает голосовые из треда. Если пользователь отвечает на чужое голосовое и обращается к боту, тот транскрибирует исходное сообщение. Транскрипт кешируется в БД — повторно за Whisper не платит.
- Админ-параметры голосовых: глобальный kill-switch `voice_enabled`, максимальная длина (`voice_max_seconds`, 0 = без лимита), дневной лимит распознаваний на чат (`whisper_daily_limit`, 0 = без лимита). Под каждым полем — справка.

### Changed

- При ошибках распознавания (Whisper недоступен, голосовое слишком длинное, исчерпан лимит) бот не отвечает канифольным «ошибка» — генерирует отговорку через LLM в текущем стиле персоны.

### Internal

- Новые сервисы `app/services/llm/whisper.py` (Whisper API) и `app/services/reply_voice.py` (reply-chain + кеш). Handler `_handle_voice_message` с engage-then-transcribe логикой. Хранение `tg_file_id` для voice/video_note без новых миграций. Правки mypy override для транзитивных импортов из тестов.

## [0.2.1] - 2026-04-14

### Internal

- Введён сервис `SpontaneityPolicy` — единая точка решения «говорить или нет». Рефакторинг интерджектора и реакций: разбросанные probability/cooldown проверки централизованы. Поведение остаётся прежним по умолчанию, но теперь после любой реплики бота включается общий cooldown — это убирает «бурстовое» поведение в часто оживающих чатах.
- В админ-панель добавлены параметры тонкой настройки спонтанности: `revive_p`, `interject_cooldown_min`, `react_cooldown_min`. Удалён устаревший `interject_cooldown` (секунды). Под полями категории появились пояснения.
- Рассылка релизов больше не уходит в приватные чаты и в группы с отключённым ботом.

## [0.2.0] - 2026-04-14

### Added

- Бот теперь видит фото из сообщения, на которое отвечают: при ответе на фото или альбом (до 10 фото) бот анализирует изображения без необходимости прикладывать их заново.

### Changed

- Автоматический fallback между LLM-провайдерами: при перегрузке основного провайдера (OpenRouter ↔ OpenAI) бот переключается на резервный и отвечает без ошибок.

### Internal

- Добавлено хранение tg_file_id и media_group_id фотографий в БД, миграция 20260413_01.
- Общий хелпер скачивания файлов Telegram в data URL.
- Рефакторинг роутера: выделены общие функции для vision-ответов.
- Правки типизации (mypy) в тестах.
