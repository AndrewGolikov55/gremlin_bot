## [0.6.0] - 2026-04-17

### Added

- Память чата: бот теперь запоминает факты об участниках (`members`) и общий контекст чата (`lore`). Когда кто-то просит запомнить факт о другом участнике — он сохраняется в память чата, а не игнорируется. Факты используются как фоновые знания — без постоянного упоминания
- Лимит: 12 записей на каждую категорию (FIFO при переполнении), каждый блок памяти занимает не более ~150 токенов
- Админ-раздел «Память участников» теперь показывает секцию «Память чата» с кнопками очистки по категориям (members, lore, всё)

### Internal

- Модель `ChatMemory`, миграция `20260417_01_chat_memory`
- Sidecar-расширение: `chat_memory_update` в JSON-схеме, парсинг, `_apply_chat_memory_update`
- Контекстный блок `build_chat_memory_block` (до 150 токенов, newest-first)
- Wiring в 3 хендлера: текстовый, фото, голосовой
- Дизайн-спек и план реализации в `docs/superpowers/`

## [0.5.0] - 2026-04-16

### Added

### Changed

### Internal

- Миграция: вайп `relationship_states`, удаление базовых персон из `style_prompts`, обновление `prompt_chat_base`
- Тесты для загрузки персон из файлов и для новой шкалы отношений
- Дизайн-спек и план реализации в `docs/superpowers/`

## [0.4.1] - 2026-04-14

### Fixed

- LLM fallback теперь срабатывает и на network errors (connection refused, DNS, timeout, reset), не только на 429/5xx. В v0.4.0 наблюдали как OpenRouter был временно недоступен на TCP-уровне — fallback на OpenAI не активировался, пользователь не получал ответ. Исправлено: если primary-провайдер вообще не отвечает, бот переключается на sibling.

## [0.4.0] - 2026-04-14

### Added

- Бот теперь может отвечать голосовыми сообщениями. По умолчанию редко (~5%), но если пользователь reply-нул боту голосовым — шанс выше (~60%, разговорный обмен голосами). Используется OpenAI TTS (`gpt-4o-mini-tts`) с opus-форматом, родным для Telegram. Применяется к direct-address, спонтанным интерджектам и revive.
- Маппинг персона → голос: каждая из 6 персон (gopnik, chatmate, standup, boss, zoomer, jarvis) озвучивается своим характерным OpenAI voice. Дефолты подобраны под персону, переопределяются в админке.
- Админ-раздел «Голосовые ответы»: kill-switch (`tts_enabled`), вероятности (`tts_reply_p` / `tts_voice_reply_p`), дневной лимит (`tts_daily_limit`), per-persona voice dropdown'ы.

### Internal

- Новые сервисы: `app/services/llm/tts.py` (OpenAI `/v1/audio/speech`), `app/bot/voice_reply.py` (`send_reply_maybe_voice` + `send_chat_maybe_voice` для voice-or-text).
- `SpontaneityPolicy.should_reply_with_voice` — decision-метод с буст-вероятностью на reply-to-bot голосовые.
- 9 новых дефолтов в `app_config` для TTS-настроек.
- При TTS-ошибке (5xx, 4xx, network) — silent fallback на текст. Voice excuses (когда Whisper не сработал) остаются текстом по дизайну.

## [0.3.2] - 2026-04-14

### Fixed

- Распознавание голосовых через Whisper: OpenAI отвергал файлы с ошибкой «Invalid file format», потому что мы отправляли их с именем `audio.bin`. Теперь имя берётся из `tg_file.file_path` (например `file_1.oga`) — OpenAI определяет формат по расширению.

## [0.3.1] - 2026-04-14

### Changed

- Бот теперь показывает статус «bot is typing...» в Telegram, пока формирует ответ. Работает во всех сценариях: прямой ответ, спонтанный комментарий, оживление молчащего чата, ответ на голосовое. Раньше пользователь видел тишину, пока бот думал.

### Internal

- Новый модуль `app/bot/typing_indicator.py` с async context manager `keep_typing`. Обернуто 6 LLM-формирующих точек: direct-reply (text/photo/voice), voice excuse, interject, revive.

## [0.3.0] - 2026-04-14

### Added

- Бот понимает голосовые сообщения и круглые видео. Задай вопрос голосовой с @упоминанием или в ответ к боту — он транскрибирует через OpenAI Whisper и ответит как на обычный текст. Плюс бот может внезапно прокомментировать чужое голосовое в чате, если сработает кубик интерджекта.
- Reply-chain: бот понимает голосовые из треда. Если пользователь отвечает на чужое голосовое и обращается к боту, тот транскрибирует исходное сообщение. Транскрипт кешируется в БД — повторно за Whisper не платит.
- Админ-параметры голосовых: глобальный kill-switch `voice_enabled`, максимальная длина (`voice_max_seconds`, 0 = без лимита), дневной лимит распознаваний на чат (`whisper_daily_limit`, 0 = без лимита). Под каждым полем — справка.

### Changed

- При ошибках распознавания (Whisper недоступен, голосовое слишком длинное, исчерпан лимит) бот не отвечает канифольным «ошибка» — генерирует отговорку через LLM в текущем стиле персоны.

### Internal

- Новые сервисы `app/services/llm/whisper.py` (Whisper API) и `app/services/reply_voice.py` (reply-chain + кеш). Handler `_handle_voice_message` с engage-then-transcribe логикой. Хранение `tg_file_id` для voice/video_note без новых миграций. Правки mypy override для транзитивных импортов из тестов.

## [0.2.1] - 2026-04-14

### Internal

- Введён сервис `SpontaneityPolicy` — единая точка решения «говорить или нет». Рефакторинг интерджектора и реакций: разбросанные probability/cooldown проверки централизованы. Поведение остаётся прежним по умолчанию, но теперь после любой реплики бота включается общий cooldown — это убирает «бурстовое» поведение в часто оживающих чатах.
- В админ-панель добавлены параметры тонкой настройки спонтанности: `revive_p`, `interject_cooldown_min`, `react_cooldown_min`. Удалён устаревший `interject_cooldown` (секунды). Под полями категории появились пояснения.
- Рассылка релизов больше не уходит в приватные чаты и в группы с отключённым ботом.

## [0.2.0] - 2026-04-14

### Added

- Бот теперь видит фото из сообщения, на которое отвечают: при ответе на фото или альбом (до 10 фото) бот анализирует изображения без необходимости прикладывать их заново.

### Changed

- Автоматический fallback между LLM-провайдерами: при перегрузке основного провайдера (OpenRouter ↔ OpenAI) бот переключается на резервный и отвечает без ошибок.

### Internal

- Добавлено хранение tg_file_id и media_group_id фотографий в БД, миграция 20260413_01.
- Общий хелпер скачивания файлов Telegram в data URL.
- Рефакторинг роутера: выделены общие функции для vision-ответов.
- Правки типизации (mypy) в тестах.
